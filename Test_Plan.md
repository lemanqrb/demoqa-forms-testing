# 🧪 DemoQA Manual Testing – Test Plan

## 🎯 1. Purpose

The purpose of this test plan is to verify the functionality of different modules on the **DemoQA** website, ensure that user interactions work as expected, and identify any functional, UI, or validation-related defects.

---

## 📂 2. Active Website Modules

- 📝 Forms
- 📦 Elements
- 🖥️ Alerts, Frame & Windows
- 🎛️ Widgets
- 🖱️ Interactions
- 📚 Book Store Application
- 🔐 Login / Register

---

## 🧪 3. Testing Types

- 🎨 UI/UX Testing

- 📏 Boundary Value Testing

## 🛠️ 4. Test Tools

- 🌐 **Google Chrome** – Executing manual test cases
- 📋 **Qase** – Creating and managing test cases, test suites, and test runs
- 🐙 **GitHub** – Documentation and version control
- 📄 **Microsoft Word** – Preparing testing documentation

---

## 💻 5. Test Environment

- **Operating System:** Windows 11 Version 25H2
- **Browser:** Google Chrome Version 146.x (Latest)
- **Website:** https://demoqa.com

---

## ✅ 6. Entry Criteria

- The DemoQA website is accessible.
- The test environment is ready.
- Test cases have been prepared.
- Required test data is available.
- The browser is installed and functioning correctly.

### 🏁 Exit Criteria

- At least **85%** of test cases have **Passed**.
- No **Critical** defects remain open.
- All planned test cases have been executed.
- The **Test Summary Report** has been completed.

---

## ⚠️ 7. Risks

- Website is unavailable.
- Unexpected system crashes.
- Internet connectivity issues.
- Browser compatibility problems.
- Changes to the website during the testing process.
- The QA Engineer is unable to complete testing within the planned timeline.
